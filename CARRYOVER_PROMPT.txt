I'm starting a new browser-based co-op roguelike game project. Here's the full context from our design discussion in another session — please read it, then help me set up the workspace.

## THE GAME — "Veins"
A 2D top-down co-op action roguelike with extraction tension, browser-only (no app stores), 2–4 players via room code. Sessions are 20–40 min runs; meta-progression persists for months.

Core innovation — the **Circulatory Board**: unlike every other co-op roguelike (Hades = solo, Gungeon = two separate guns, RoR2 = items don't interact between players), the party SHARES one hexagonal relic board. Relics only fire their strongest effect when adjacent to a teammate's compatible relic. The build space is COMBINATORIAL across players, not additive — you optimize the party organism, not your own loadout. "A roguelike you literally cannot beat by yourself."

Two supporting mechanics:
- **Bleed Clock**: the dungeon has a global HP bar draining in real time, faster the deeper you go. Loot scales with depth → constant "extract now or push one more floor?" group tension (FTL-style dread, but shared and vocal).
- **Linked Fates**: reviving a downed teammate costs one of YOUR board relics, sacrificed into their slot — death reshapes the party build mid-fight.

## TECH DECISIONS (already made)
- Renderer: Phaser.js (WebGL), 2D top-down twin-stick (WASD + mouse). 3D rejected — costs load time, asset budget, network payload for no benefit.
- Netcode: AUTHORITATIVE server (source of truth), clients render delta updates only. Room-based + forgiving hit windows make ~50–80ms RTT imperceptible. Start here, not client-prediction.
- Dungeon gen: seeded BSP tree (<5ms, deterministic from run ID — enables daily-challenge + bug repro).
- Pathfinding: A* on grid. Collision: spatial hashing (O(n²) → O(1) avg).
- Loot/synergy: weighted random tables with tags (e.g. [fire, aoe, party]).
- ALL procedural logic runs server-side (anti-cheat, thin clients).
- Frontend UI/lobby/menus: React + Socket.io client.

## STACK & DEPLOYMENT (no-cost priority)
- Frontend (React + Phaser): Vercel free tier.
- Backend (Node + Socket.io authoritative server): Fly.io free tier (stays alive on WebSockets, unlike Render which spins down). ~256MB handles 50–100 concurrent players in separate rooms.
- DB (meta-progression / unlocks / auth ONLY): Supabase free tier. Rooms are ephemeral/in-memory, never persisted.
- Cost stays $0 until hundreds of concurrent players.

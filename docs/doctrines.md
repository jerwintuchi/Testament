This file is the *concept* of doctrine. For the four belief-systems as in-world factions/bosses/aesthetics, see [factions.md](factions.md). For the hidden scoring implementation, see [systems/doctrine-tracking.md](systems/doctrine-tracking.md).

## Build = Belief

Players unconsciously align with doctrines based on playstyle. A doctrine is *behavior under stress*, expressed through how the party builds the Circulatory Board.

> Build = Doctrine · Doctrine = Behavior under stress · Behavior = World interpretation

Players do not choose classes. They reveal beliefs under pressure.

## The Four Axes

Each run trends along four belief axes (detailed identities in [factions.md](factions.md)):

- **Sanctum** — stability / order. *"Truth is stable form."*
- **Tumorous Host** — mutation / chaos. *"Truth is constant becoming."*
- **Synaptic Chorus** — unity / synchronization. *"Truth is collective mind."*
- **Penitent** — sacrifice / withdrawal. *"Truth is surrender."*

## Relics as Doctrine Vocabulary

Relics are not items. They are:

> Arguments about how reality works.

Each relic belongs loosely to a doctrine but can mix. Relic categories by doctrine are catalogued in [content/relic-roster.md](content/relic-roster.md); the relic *system* is in [systems/relics.md](systems/relics.md).

## Relic Adjacency = Theology Syntax

Relic placement forms "sentences of belief":

- **Linear chains** → Sanctum logic
- **Web clusters** → Tumor logic
- **Mirror pairs** → Chorus logic
- **Isolation nodes** → Penitent logic

> Players unknowingly write ideology through builds.

## No explicit doctrine UI

Doctrine alignment is **never shown numerically**. Players never see "Sanctum +12" — only effects manifest. This is a non-negotiable design principle (see [vision.md](vision.md)) and shapes the HUD (see [ui-style-guide.md](ui-style-guide.md)). The system reacts with delay, reinforcing the illusion that the world *remembers* and *interprets*.
